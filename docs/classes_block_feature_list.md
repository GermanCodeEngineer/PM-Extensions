# Classes Extension Feature Correlation List

## Key Terms

- scope: A visibility layer for variables. Inner scope means the most local active layer; outer scope means a parent layer. Examples: "create local variable scope", "create function at var", "create class at var".
- shadowing: A variable in an inner scope uses the same name as one in an outer scope, so the inner one is used first.
- class definition context: The time and scope where a class is being defined (for example inside "create class at var", "create class named", or "on class").
- inheritance: A class-child relation where a subclass can use members (for example methods or class variables) from a parent class.
- call routing: How a call goes to the correct method implementation.
- accessor: A getter or setter method used instead of direct attribute read/write.
- identity: Whether two values are the exact same object, not just equal-looking values.
- special no-value: A dedicated value used to represent "no result"; here this is "Nothing".
A
- Feature: How variables behave across scopes
  - Blocks involved
    - "set var [NAME] to [VALUE] in current scope"
    - "get var [NAME]"
    - "var [NAME] exists in [KIND]?"
    - "delete var [NAME] in current scope"
    - "create local variable scope"
    - "all variables in [KIND]"
  - Behavior details
    - When a name exists in more than one scope, the inner scope value is used first.
    - When a local scope ends, local variables disappear, but outer variables stay.
    - "all variables in" lists names based on the selected scope filter and avoids duplicate shadowed names.
    - While defining a class, variable set and delete behavior is limited on purpose.

- Feature: Linked variables between scopes
  - Blocks involved
    - "bind [KIND] variable [NAME] to current scope"
    - "set var [NAME] to [VALUE] in current scope"
    - "get var [NAME]"
    - "create local variable scope"
  - Behavior details
    - "bind variable to current scope" links to the same variable, not a copy.
    - If you change the linked variable in an inner scope, the source value also changes.
    - If a linked variable is deleted, it disappears everywhere that shared link is used.
    - Linking fails if the source variable does not exist in the selected origin.

- Feature: Functions and methods keep outer context safely
  - Blocks involved
    - "create function at var [NAME]"
    - "create function named [NAME]"
    - "define instance method [NAME] [SHADOW]"
    - "define static method [NAME]"
    - "return [VALUE]"
  - Behavior details
    - Functions and methods can use outer variables from where they were defined.
    - "return" block exits the current function or method cleanly.
    - "return" only works while a function or method call is active; using it elsewhere fails.
    - If execution fails, temporary inner scopes are cleaned up instead of leaking.

- Feature: Function arguments and defaults
  - Blocks involved
    - "configure next function: argument names [ARGNAMES] defaults [ARGDEFAULTS]"
    - "create function at var [NAME]"
    - "create function named [NAME]"
    - "call function [FUNC] with positional args [POSARGS]"
  - Behavior details
    - Positional inputs are matched to configured argument names.
    - Default values are used when trailing values are missing.
    - Too many arguments or missing required ones cause an error.
    - Invalid setup (more defaults than names) also causes an error.

- Feature: Create classes now, extend classes later
  - Blocks involved
    - "create class at var [NAME] [SHADOW]"
    - "create class named [NAME] [SHADOW]"
    - "on class [CLASS] [SHADOW]"
    - "class being created"
  - Behavior details
    - You can define a class in one step or add more members later with "on class".
    - "class being created" works only while class definition is active

- Feature: Inheritance behavior
  - Blocks involved
    - "create subclass at var [NAME] with superclass [SUPERCLASS] [SHADOW]"
    - "create subclass named [NAME] with superclass [SUPERCLASS] [SHADOW]"
    - "is [SUBCLASS] a subclass of [SUPERCLASS] ?"
    - "get superclass of [CLASS]"
  - Behavior details
    - "is a subclass of ?" checks the full parent chain, not just the direct parent.
    - "get superclass of" returns "Nothing" when a class has no parent.

- Feature: How class members are found and validated
  - Blocks involved
    - "define instance method [NAME] [SHADOW]"
    - "define static method [NAME]"
    - "define getter [NAME] [SHADOW]"
    - "define setter [NAME] [SHADOW1] [SHADOW2]"
    - "define operator method [OPERATOR_KIND] [SHADOW]"
    - "on [CLASS] set class variable [NAME] to [VALUE]"
    - "get class variable [NAME] of [CLASS]"
    - "on [CLASS] delete class variable [NAME]"
    - "[PROPERTY] names of class [CLASS]"
  - Behavior details
    - Local class members are used first, then inherited ones.
    - Subclass members replace same-name inherited members.
    - A getter and setter can share one name, but invalid redefinitions are blocked.
    - Class variables can be set, read, listed, and deleted as class-level metadata.
    - Deleting by member type checks both existence and matching type.

- Feature: Instance creation rules
  - Blocks involved
    - "create instance of class [CLASS] with positional args [POSARGS]"
    - "define [SPECIAL_METHOD] instance method [SHADOW]"
    - "call super init method with positional args [POSARGS]"
    - "return [VALUE]"
    - "Nothing"
  - Behavior details
    - `init` runs when an instance is created.
    - `init` must return "Nothing", otherwise it fails.
    - A subclass can call parent init with "call super init method with positional args".

- Feature: Calling methods and parent methods
  - Blocks involved
    - "on [INSTANCE] call method [NAME] with positional args [POSARGS]"
    - "call super method [NAME] with positional args [POSARGS]"
    - "self"
    - "define instance method [NAME] [SHADOW]"
  - Behavior details
    - "on call method with positional args" calls an instance method by name.
    - "self" is only available while a method-like call is active.
    - "call super method with positional args" calls the parent version even if the subclass overrides it.
    - Calling a super method fails if there is no parent class.

- Feature: Attribute read and write behavior
  - Blocks involved
    - "on [INSTANCE] set attribute [NAME] to [VALUE]"
    - "attribute [NAME] of [INSTANCE]"
    - "all attributes of [INSTANCE]"
    - "define getter [NAME] [SHADOW]"
    - "define setter [NAME] [SHADOW1] [SHADOW2]"
    - "value"
    - "self"
  - Behavior details
    - Plain attributes work directly if no getter or setter is defined.
    - A getter can run instead of a direct read.
    - A setter can run instead of a direct write, and can read the incoming value via "value".
    - The reporter "value" only works while a setter is running.
    - Writing to a getter-only attribute fails.

- Feature: Custom operator behavior
  - Blocks involved
    - "define operator method [OPERATOR_KIND] [SHADOW]"
    - "other value"
    - "return [VALUE]"
    - "self"
  - Behavior details
    - "other value" works only inside an operator method.

- Feature: Static methods and class-level calls
  - Blocks involved
    - "define static method [NAME]"
    - "on [CLASS] call static method [NAME] with positional args [POSARGS]"
    - "get static method [NAME] of [CLASS] as function"
    - "call function [FUNC] with positional args [POSARGS]"
  - Behavior details
    - Static methods can be inherited from parent classes.
    - "get static method of as function" lets you pass a static method around as a function value.

- Feature: Typed inputs resolve from values or variable names
  - Blocks involved
    - "call function [FUNC] with positional args [POSARGS]"
    - "create instance of class [CLASS] with positional args [POSARGS]"
    - "on [CLASS] call static method [NAME] with positional args [POSARGS]"
    - "on [INSTANCE] call method [NAME] with positional args [POSARGS]"
  - Behavior details
    - Inputs that expect a function, class, or instance can use either the value itself or the name of a variable holding that value.
    - A missing name or a value of the wrong kind causes an error instead of silently continuing.

- Feature: Type checks, identity, and special no-value
  - Blocks involved
    - "is [INSTANCE] an instance of [CLASS] ?"
    - "get class of [INSTANCE]"
    - "[VALUE1] is [VALUE2] ?"
    - "typeof [VALUE]"
    - "[VALUE] as string"
    - "Nothing"
  - Behavior details
    - "is an instance of ?" respects inheritance (subclass instance counts as base instance).
    - "is ?" checks whether two values are the same exact object.
    - "as string" gives stable readable text for classes, instances, and `Nothing`; class text includes superclass info when present.
    - "Nothing" is a stable special no-value. When it is converted to a boolean it evaluates to `false`.

- Feature: Introspection and listing helpers
  - Blocks involved
    - "[PROPERTY] names of class [CLASS]"
    - "all attributes of [INSTANCE]"
    - "all variables in [KIND]"
  - Behavior details
    - "names of class" can list specific member categories (instance, static, getter, setter, operator, class variable).
    - These listing blocks make it easier to inspect and test current class or instance state.
